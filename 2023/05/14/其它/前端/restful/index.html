Restful 接口

设计项目资源的 URL

资源定位

资源集合用一个 URL ，具体某个资源用一个 URL。

资源请求

使用四种 HTTP 方法可以提供CRUD功能：

  • 获取：使用GET方法获取资源。
  • 创建：使用POST创建新的资源。
  • 更新：使用PUT更新现有资源。（和 POST 近似）
  • 删除：使用DELETE删除现有资源。（和 GET 近似）

请求参数

可选的、复杂的参数用查询字符串表示。
使用小驼峰命名法作为属性标识符。

返回状态码

RESTful Web服务应使用合适的HTTP状态码来响应客户端请求

2xx - 成功 - 一切都很好
4xx - 客户端错误 - 如果客户端发生错误（例如客户端发送无效请求或未被授权）
5xx – 服务器错误 - 如果服务器发生错误（例如，尝试处理请求时出错）
